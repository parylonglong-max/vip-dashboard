# 设计规范 — 精品货价监控数据看板

## 1. 设计理念

**蓝白拟态风（Blue-White Neumorphism）**

以柔和的蓝白灰色调为基底，通过双向阴影（凸起/凹陷）营造层次感和触感，强调数据的清晰直观，避免过度装饰。整体风格简洁、专业、手机友好。

### 关键词
- 柔和、干净、专业
- 触感反馈（拟态凸起→按下凹陷）
- 数据为主、装饰为辅
- 手机优先、单手操作

## 3. 拟态阴影系统

### 凸起态（默认）
```css
box-shadow:
  8px 8px 20px rgba(163, 190, 220, 0.4),
  -8px -8px 20px rgba(255, 255, 255, 0.9);
```

### 凸起态-小
```css
box-shadow:
  4px 4px 10px rgba(163, 190, 220, 0.35),
  -4px -4px 10px rgba(255, 255, 255, 0.85);
```

### 凹陷态
```css
box-shadow:
  inset 4px 4px 10px rgba(163, 190, 220, 0.35),
  inset -4px -4px 10px rgba(255, 255, 255, 0.85);
```

### 凹陷态-小
```css
box-shadow:
  inset 2px 2px 6px rgba(163, 190, 220, 0.3),
  inset -2px -2px 6px rgba(255, 255, 255, 0.8);
```

**使用规则：**
- 卡片、按钮默认 = 凸起
- 输入框、激活态切换 = 凹陷
- 按下（:active）= 凸起→凹陷过渡

## 6. 移动端布局

### 布局结构

```
┌─────────────────────┐
│   粘性导航栏 (sticky) │
├─────────────────────┤
│  MTD / YTD 切换按钮   │
├─────────────────────┤
│  汇总概览条           │
├─────────────────────┤
│  ┌─ 模块1 (可折叠) ─┐ │
│  │ 汇总行            │
│  │ 小组A 表格         │
│  │ 小组B 表格         │
│  └─────────────────┘ │
│  ┌─ 模块2 (可折叠) ─┐ │
│  │ ...              │ │
│  └─────────────────┘ │
└─────────────────────┘
```

### 关键设计决策

1. **单列布局**：手机端不需要多列，所有数据纵向排列
2. **粘性导航**：滚动时保持标题和退出按钮可见
3. **折叠模块**：默认展开，点击标题折叠/展开，减少滚动疲劳
4. **表格自适应**：表格列宽自动分配，指标列左对齐，数值列右对齐
5. **触控区域**：所有可点击元素 ≥ 40px 高度

### 响应式断点

| 断点 | 容器宽度 | 调整内容 |
|------|---------|---------|
| < 360px | 100% | 缩小字号、紧凑间距 |
| 360-767px | 100% | 标准手机布局 |
| 768-1023px | 720px 居中 | 放大字号 |
| ≥ 1024px | 900px 居中 | 表格行高加大 |

## 7. 组件清单

### 7.1 登录卡片 (LoginCard)
- 圆角拟态卡片居中
- 图标 + 标题 + 密码输入 + 按钮
- 错误时抖动动画

### 7.2 导航栏 (Navbar)
- sticky 定位
- 左：logo + 标题
- 右：更新日期 + 退出按钮

### 7.3 周期切换 (PeriodToggle)
- 拟态凹陷容器 + 凸起激活态
- MTD / YTD 两个按钮
- 切换时重渲染所有模块

### 7.4 汇总概览 (Overview)
- 三列横排：实际 / 目标 / 达成率
- 达成率带状态颜色

### 7.5 模块卡片 (Module)
- 可折叠容器
- 头部：图标 + 名称 + 折叠箭头
- 正文：汇总行 + 小组表格

### 7.6 小组表格 (GroupTable)
- 小组名称标签
- 4 列表格：指标 | 实际 | 目标 | 达成/同比
- 状态颜色标签

### 7.7 状态标签 (StatusBadge)
- 圆角小标签
- 4 种状态：up / down / warning / neutral
- 同时显示达成率 + 同比

## 8. 交互规范

| 操作 | 反馈 |
|------|------|
| 点击登录按钮 | 验证密码，成功→看板，失败→抖动+错误提示 |
| 点击 MTD/YTD | 切换所有数据，激活态视觉变化 |
| 点击模块标题 | 折叠/展开模块内容 |
| 点击退出 | 返回登录页，清空状态 |
| 输入密码回车 | 等同点击登录按钮 |

### 过渡动画
- 通用过渡：`0.25s ease`
- 折叠箭头旋转：`0.3s ease`
- 按钮按下：凸起→凹陷（即时）
- 登录失败抖动：`0.4s ease`

## 9. 特殊处理

### "越低越好"指标

部分指标（如折扣率、周转天数）数值越低越好，系统会自动反算达成率：

```javascript
var lowerIsBetter = metric.indexOf("折扣") > -1 ||
                    metric.indexOf("周转") > -1;
// 达成率 = target / actual（而非 actual / target）
```

### 百分比指标

指标名含 `(%)` 或 `率` 的自动加 `%` 后缀。
